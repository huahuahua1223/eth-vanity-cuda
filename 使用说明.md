# ETH Vanity Address 使用说明

## 功能更新

现在程序支持自动将结果保存到文件，文件名根据前缀和后缀中9的数量自动生成。

## 使用方法

### 1. 单次运行（自动命名文件）

程序会自动计算前缀和后缀中连续9的数量，并生成对应的文件名。

```bash
# 示例：前缀3个9，后缀3个9
# 会自动生成文件：prefix_3_suffix_3.txt
./eth-vanity-address --contract --device 0 --prefix 999 --suffix 999

# 示例：前缀5个9，后缀3个9
# 会自动生成文件：prefix_5_suffix_3.txt
./eth-vanity-address --contract --device 0 --prefix 99999 --suffix 999
```

### 2. 手动指定输出文件

如果你想自己指定文件名：

```bash
./eth-vanity-address --contract --device 0 --prefix 999 --suffix 999 --output my_results.txt
```

## 输出文件格式

生成的文件包含以下信息：
```
Elapsed: 000002 Score: 01 Private Key: 0x... Address: 0x999...999
```

- **Elapsed**: 运行时间（秒）
- **Score**: 评分
- **Private Key**: 私钥（用于生成合约地址）
- **Address**: 生成的合约地址

## 示例输出文件

假设你运行从前缀3个9到9个9，后缀都是3个9，会生成以下文件：

- `prefix_3_suffix_3.txt` - 前缀999，后缀999的地址
- `prefix_4_suffix_3.txt` - 前缀9999，后缀999的地址
- `prefix_5_suffix_3.txt` - 前缀99999，后缀999的地址
- `prefix_6_suffix_3.txt` - 前缀999999，后缀999的地址
- `prefix_7_suffix_3.txt` - 前缀9999999，后缀999的地址
- `prefix_8_suffix_3.txt` - 前缀99999999，后缀999的地址
- `prefix_9_suffix_3.txt` - 前缀999999999，后缀999的地址

## 注意事项

1. 随着9的数量增加，找到匹配地址的难度会**指数级增长**
2. 文件会以**追加模式**打开，不会覆盖已有内容
3. 程序运行时会实时写入文件并刷新，不用担心数据丢失
4. 可以随时按 Ctrl+C 停止程序

## 性能提示

- 前缀3-4个9：通常几秒到几分钟就能找到
- 前缀5-6个9：可能需要几分钟到几小时
- 前缀7-8个9：可能需要几小时到几天
- 前缀9个9：可能需要很长时间
